プロジェクトを削除すると配下のタスクも全て削除されるため、確認ダイアログを挟みます。

このプロジェクトに紐づく 2 件のタスク も全て削除されます。この操作は取り消せません。

仕様メモ

  • 追加: フォームに入力 → 「追加」クリックで作成。重複名チェックなし (UI 上では区別しないが内部 ID で管理)。
  • 編集: 「編集」ボタンで Input に切り替え。「保存」or Enter で確定、「キャンセル」or Esc で破棄。
  • 削除: 「削除」ボタン → 確認ダイアログ → 「削除する」で `DELETE /api/projects/:id`。 FK CASCADE で配下タスクも自動削除。
  • 並び替え: ホバーで左端のドラッグハンドルが現れる。D&D で順序変更 → `PUT /api/projects/reorder`。
  • 件数表示: 各行の右側に「N 件のタスク」を表示。0 件のときは「タスクなし」(薄色)。
  • 空状態: プロジェクト 0 件のときは「プロジェクトがまだありません」と表示。
  • 長期タスクの分類: このタブでは扱わない (現行どおり長期タブ内で完結)。
